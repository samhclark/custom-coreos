## List of exported metrics

### Garage system metrics

#### `garage_build_info` (counter) {#garage-build-info-counter}

Exposes the Garage version number running on a node.

``` {style="background-color:#2b303b;color:#c0c5ce;"}
garage_build_info{version="1.0"} 1
```

#### `garage_replication_factor` (counter) {#garage-replication-factor-counter}

Exposes the Garage replication factor configured on the node

``` {style="background-color:#2b303b;color:#c0c5ce;"}
garage_replication_factor 3
```

#### `garage_local_disk_avail` and `garage_local_disk_total` (gauge) {#garage-local-disk-avail-and-garage-local-disk-total-gauge}

Reports the available and total disk space on each node, for data and
metadata separately.

``` {style="background-color:#2b303b;color:#c0c5ce;"}
garage_local_disk_avail{volume="data"} 540341960704
garage_local_disk_avail{volume="metadata"} 540341960704
garage_local_disk_total{volume="data"} 763063566336
garage_local_disk_total{volume="metadata"} 763063566336
```

### Cluster health status metrics

#### `cluster_healthy` (gauge) {#cluster-healthy-gauge}

Whether all storage nodes are connected (0 or 1)

``` {style="background-color:#2b303b;color:#c0c5ce;"}
cluster_healthy 0
```

#### `cluster_available` (gauge) {#cluster-available-gauge}

Whether all requests can be served, even if some storage nodes are
disconnected

``` {style="background-color:#2b303b;color:#c0c5ce;"}
cluster_available 1
```

#### `cluster_connected_nodes` (gauge) {#cluster-connected-nodes-gauge}

Number of nodes currently connected

``` {style="background-color:#2b303b;color:#c0c5ce;"}
cluster_connected_nodes 3
```

#### `cluster_known_nodes` (gauge) {#cluster-known-nodes-gauge}

Number of nodes already seen once in the cluster

``` {style="background-color:#2b303b;color:#c0c5ce;"}
cluster_known_nodes 3
```

#### `cluster_layout_node_connected` (gauge) {#cluster-layout-node-connected-gauge}

Connection status for individual nodes of the cluster layout

``` {style="background-color:#2b303b;color:#c0c5ce;"}
cluster_layout_node_connected{id="62b218d848e86a64",role_capacity="1000000000",role_gateway="0",role_zone="dc1"} 1
cluster_layout_node_connected{id="a11c7cf18af29737",role_capacity="1000000000",role_gateway="0",role_zone="dc1"} 0

cluster_layout_node_connected{id="b10c110e4e854e5a",role_capacity="1000000000",role_gateway="0",role_zone="dc1"} 1
```

#### `cluster_layout_node_disconnected_time` (gauge) {#cluster-layout-node-disconnected-time-gauge}

Time (in seconds) since last connection to individual nodes of the
cluster layout

``` {style="background-color:#2b303b;color:#c0c5ce;"}
cluster_layout_node_disconnected_time{id="62b218d848e86a64",role_capacity="1000000000",role_gateway="0",role_zone="dc1"} 0
cluster_layout_node_disconnected_time{id="a235ac7695e0c54d",role_capacity="1000000000",role_gateway="0",role_zone="dc1"} 0
cluster_layout_node_disconnected_time{id="b10c110e4e854e5a",role_capacity="1000000000",role_gateway="0",role_zone="dc1"} 0
```

#### `cluster_storage_nodes` (gauge) {#cluster-storage-nodes-gauge}

Number of storage nodes declared in the current layout

``` {style="background-color:#2b303b;color:#c0c5ce;"}
cluster_storage_nodes 4
```

#### `cluster_storage_nodes_ok` (gauge) {#cluster-storage-nodes-ok-gauge}

Number of storage nodes currently connected

``` {style="background-color:#2b303b;color:#c0c5ce;"}
cluster_storage_nodes_ok 3
```

#### `cluster_partitions` (gauge) {#cluster-partitions-gauge}

Number of partitions in the layout (this is always 256)

``` {style="background-color:#2b303b;color:#c0c5ce;"}
cluster_partitions 256
```

#### `cluster_partitions_all_ok` (gauge) {#cluster-partitions-all-ok-gauge}

Number of partitions for which all storage nodes are connected

``` {style="background-color:#2b303b;color:#c0c5ce;"}
cluster_partitions_all_ok 64
```

#### `cluster_partitions_quorum` (gauge) {#cluster-partitions-quorum-gauge}

Number of partitions for which we have a quorum of connected nodes and
all requests can be served

``` {style="background-color:#2b303b;color:#c0c5ce;"}
cluster_partitions_quorum 256
```

### Metrics of the API endpoints

#### `api_admin_request_counter` (counter) {#api-admin-request-counter-counter}

``` {style="background-color:#2b303b;color:#c0c5ce;"}
api_admin_request_counter{api_endpoint="Metrics"} 127041
```

#### `api_admin_request_duration` (histogram) {#api-admin-request-duration-histogram}

Evaluates the duration of API calls to the various administration API
endpoint. Example:

``` {style="background-color:#2b303b;color:#c0c5ce;"}
api_admin_request_duration_bucket{api_endpoint="Metrics",le="0.5"} 127041
api_admin_request_duration_sum{api_endpoint="Metrics"} 605.250344830999
api_admin_request_duration_count{api_endpoint="Metrics"} 127041
```

#### `api_s3_request_counter` (counter) {#api-s3-request-counter-counter}

Counts the number of requests to a given endpoint of the S3 API.
Example:

``` {style="background-color:#2b303b;color:#c0c5ce;"}
api_s3_request_counter{api_endpoint="CreateMultipartUpload"} 1
```

#### `api_s3_error_counter` (counter) {#api-s3-error-counter-counter}

Counts the number of requests to a given endpoint of the S3 API that
returned an error. Example:

``` {style="background-color:#2b303b;color:#c0c5ce;"}
api_s3_error_counter{api_endpoint="GetObject",status_code="404"} 39
```

#### `api_s3_request_duration` (histogram) {#api-s3-request-duration-histogram}

Evaluates the duration of API calls to the various S3 API endpoints.
Example:

``` {style="background-color:#2b303b;color:#c0c5ce;"}
api_s3_request_duration_bucket{api_endpoint="CreateMultipartUpload",le="0.5"} 1
api_s3_request_duration_sum{api_endpoint="CreateMultipartUpload"} 0.046340762
api_s3_request_duration_count{api_endpoint="CreateMultipartUpload"} 1
```

### Metrics of the data block manager

#### `block_bytes_read`, `block_bytes_written` (counter) {#block-bytes-read-block-bytes-written-counter}

Number of bytes read/written to/from disk in the data storage directory.

``` {style="background-color:#2b303b;color:#c0c5ce;"}
block_bytes_read 120586322022
block_bytes_written 3386618077
```

#### `block_ram_buffer_free_kb` (gauge) {#block-ram-buffer-free-kb-gauge}

Kibibytes available for buffering blocks that have to be sent to remote
nodes. When clients send too much data to this node and a storage node
is not receiving data fast enough due to slower network conditions, this
will decrease down to zero and backpressure will be applied.

``` {style="background-color:#2b303b;color:#c0c5ce;"}
block_ram_buffer_free_kb 219829
```

#### `block_compression_level` (counter) {#block-compression-level-counter}

Exposes the block compression level configured for the Garage node.

``` {style="background-color:#2b303b;color:#c0c5ce;"}
block_compression_level 3
```

#### `block_read_duration`, `block_write_duration` (histograms) {#block-read-duration-block-write-duration-histograms}

Evaluates the duration of the reading/writing of individual data blocks
in the data storage directory.

``` {style="background-color:#2b303b;color:#c0c5ce;"}
block_read_duration_bucket{le="0.5"} 169229
block_read_duration_sum 2761.6902550310056
block_read_duration_count 169240
block_write_duration_bucket{le="0.5"} 3559
block_write_duration_sum 195.59170078500006
block_write_duration_count 3571
```

#### `block_delete_counter` (counter) {#block-delete-counter-counter}

Counts the number of data blocks that have been deleted from storage.

``` {style="background-color:#2b303b;color:#c0c5ce;"}
block_delete_counter 122
```

#### `block_resync_counter` (counter), `block_resync_duration` (histogram) {#block-resync-counter-counter-block-resync-duration-histogram}

Counts the number of resync operations the node has executed, and
evaluates their duration.

``` {style="background-color:#2b303b;color:#c0c5ce;"}
block_resync_counter 308897
block_resync_duration_bucket{le="0.5"} 308892
block_resync_duration_sum 139.64204196100016
block_resync_duration_count 308897
```

#### `block_resync_queue_length` (gauge) {#block-resync-queue-length-gauge}

The number of block hashes currently queued for a resync. This is normal
to be nonzero for long periods of time.

``` {style="background-color:#2b303b;color:#c0c5ce;"}
block_resync_queue_length 0
```

#### `block_resync_errored_blocks` (gauge) {#block-resync-errored-blocks-gauge}

The number of block hashes that we were unable to resync last time we
tried. **THIS SHOULD BE ZERO, OR FALL BACK TO ZERO RAPIDLY, IN A HEALTHY
CLUSTER.** Persistent nonzero values indicate that some data is likely
to be lost.

``` {style="background-color:#2b303b;color:#c0c5ce;"}
block_resync_errored_blocks 0
```

### Metrics related to RPCs (remote procedure calls) between nodes
